ESP32 IMU Wireless Remote

Project Repository: github.com/jacobagerson/ESP32_KiCAD_IMU

Synth-esthesia is an electronic musical instrument that mimics the experience of synesthesia, a nuerological condition where stimulation of one sensory pathway prompts automatic experiences in a second sensory pathway. Synth-esthesia enables users to interact primarily visually with the painting interface using the wireless paintbrush, and explore a sonic atmosphere by discovering which sounds and effects correspond to which visual cues, emulating the experience of synesthesia. I was the sole hardware engineer on the 3-person team building the instrument, and was responsible for designing and fabricating the custom ESP32 IMU wireless remote that serves as the instrument's primary input device!


Below is my part of our group's final project presentation. I describe the broad technical specifications of the remote, and the role it plays as a part of the instrument!

Project Description

The ESP32 IMU wireless remote is a custom-designed PCB built around an ESP32-WROOM-32 microcontroller and an InvenSense MPU-6050 IMU. The remote captures motion data from the IMU, processes it, and transmits the resulting orientation data wirelessly over Wi-Fi to the Synth-esthesia painting interface. The remote is powered by a rechargeable LiPo battery, and includes power management circuitry ensuring ~1 hour of use time between charges. The PCB was designed using KiCAD, fabricated by JLCPCB, and assembled by hand in Tufts' Halligan Labs!

Key features of the ESP32 IMU wireless remote include:

  • ESP32-WROOM-32 microcontroller for Wi-Fi connectivity and processing power
  • InvenSense MPU-6050 6-axis IMU for capturing motion data
  • Custom sensor fusion and calibration algorithms to compute orientation from raw IMU data
  • LiPo battery power with charging and power management circuitry
  • Compact PCB design optimized for handheld use

5 Stage Pipelined ARM CPU in VHDL

Project Repository: github.com/jacobagerson/EE25_Pipelined_CPU

Designed and implemented a 5-stage pipelined ARM CPU architecture in VHDL as part of an advanced digital design course. The processor implements a subset of the ARM instruction set with a classical 5-stage pipeline (Fetch, Decode, Execute, Memory, Write-back) optimized for performance and resource efficiency on FPGA.

Key Features

  • 5-stage pipeline architecture for increased throughput
  • Subset of ARM instruction set support (data processing, memory operations, branches)
  • Forwarding and hazard detection logic to minimize pipeline stalls
  • 16-bit data width with 32-bit program counter
  • Implemented in VHDL and synthesized on Xilinx FPGA
  • Full testbench with comprehensive test vectors

Architecture Overview

The pipeline stages include instruction fetching from ROM, instruction decoding with register file access, ALU execution, memory read/write operations, and register write-back. The design includes hazard detection for data dependencies and control flow changes.

Embedded Smart Watch Mini-Golf Game

Project Repository: github.com/jacobagerson/EE14_Golf_Game

Developed an interactive mini-golf game running on an embedded STM32 Nucleo microcontroller with a 1.69-inch LCD display. The game uses an MPU-6050 6-axis IMU to detect device motion and tilt for controlling the golf club angle and ball trajectory. Players tilt the smartwatch to aim and swing at virtual golf balls on various challenging courses.

Hardware Components

  • STM32 Nucleo-L476RG microcontroller
  • InvenSense MPU-6050 6-axis IMU for motion detection
  • 1.69-inch 240x280 pixel LCD display (ST7789 driver)
  • Li-ion battery with charging circuitry
  • Custom 3D-printed enclosure

Software Features

  • Real-time IMU sensor fusion algorithms for accurate tilt detection
  • Physics-based golf ball simulation with gravity and friction
  • Multiple challenging golf courses with obstacles
  • Score tracking and leaderboard functionality
  • Touch-optimized UI for small display

Chrome Dinosaur Game Hardware Accelerator in VHDL

Project Repository: github.com/jacobagerson/Jumbo_Dash

Implemented a hardware-accelerated version of Chrome's offline dinosaur game on an Upduino ICE40UP5K FPGA. The game logic and animation are implemented in VHDL with VGA output to a standard monitor. Players use an old-school NES controller to jump, crouch and navigate the start and end screens!


Watch me playing Jumbo Dash during our final project presentation below!

Hardware Specifications

  • Upduino 3.1 development board with Lattice ICE40UP5K FPGA
  • VGA output (640x480 @ 60Hz) to standard monitor
  • 2 push buttons for game input (jump and crouch controls)
  • Implemented entirely in VHDL for hardware acceleration

Game Features

  • Real-time sprite rendering and collision detection
  • Progressive difficulty with increasing obstacle speed
  • Score counter and high-score tracking in FPGA memory
  • Physics-based jump mechanics with gravity simulation
  • Multiple obstacle types and terrain variations

Technical Implementation

The design features a VGA controller for display output, sprite graphics engine for rendering game objects, and a state machine for game logic. Collision detection is performed in real-time using pixel-perfect algorithms. The entire design was synthesized and placed-and-routed on the ICE40UP5K, utilizing approximately 40% of available logic resources.
